# EducaOne v2.13.32 — Backups corregidos + listo para desplegar

## Cambio en esta versión

### backup_scheduler interno DESACTIVADO (a propósito)
El sistema tenía un scheduler que hacía backups cada día a las 2 AM y los
guardaba en el disco del servidor. PROBLEMA: en Render el disco es efímero
(se borra en cada redeploy/reinicio) → esos backups se perdían y daban
una FALSA sensación de seguridad.

Lo desactivé. El código queda comentado (reversible), pero ya no corre.

### La estrategia CORRECTA de backups (lo que sí funciona)
1. Backups automáticos de Render PostgreSQL (plan de pago): confiables,
   persisten, se restauran con un clic desde el dashboard de Render.
2. Opcional/adicional: Cron Job de Render que ejecute backup_datos.py
   (incluido) y suba el JSON a Google Drive o S3.

## Cómo entrar la primera vez en producción (primer login)
Al arrancar con base vacía, init_db() crea automáticamente:
- Usuario 'superadmin' con password ALEATORIA que aparece en los LOGS de
  Render (bloque "SUPERADMIN INICIAL CREADO"). Copiala de ahí.
- Usuario 'direccion' con password 'admin123' (te obliga a cambiarla en
  producción al primer login).
NO te quedás trabado: el sistema se autoconfigura (tablas + seed).
